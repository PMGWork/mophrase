# 手動回帰チェックリスト

## 使い方

1. 差分に関連するセクションだけ実施する。  
2. 各項目を `PASS / FAIL / SKIP` で記録する。  
3. `FAIL` は必ず再現手順を1行残す。

## A. Pen Tool

- `Pen` を選択してドラッグし、新規パスが作成される。
- マウスリリース後、キーフレームが2点以上のときだけ確定する。
- 連続で2本以上描いても、既存パスの表示が崩れない。

## B. Select Tool

- `Select` を選択し、パスをクリックしてアクティブ化できる。
- 背景クリックで選択解除される。
- ドラッグで範囲選択し、選択範囲に応じて提案UI位置が更新される。

## C. Handle Drag

- アンカー/ハンドルをドラッグして形状が更新される。
- `Alt` 押下時に自由ドラッグ、非押下時にミラー挙動になる。
- ドラッグ終了後、意図しないハンドル選択が残り続けない。

## D. Graph Editor

- アクティブパスがあるとグラフが描画される。
- グラフハンドルをドラッグしてイージング形状が変わる。
- `Alt` の同期解除が効き、片側だけの調整ができる。

## E. Suggestion UI

- `Select` + パス選択で提案入力UIが表示される。
- 送信中は `Generating...` が表示される。
- 提案ホバーでプレビューが反映され、クリックで適用される。
- APIが使えない環境では `SKIP` として理由を記録する。

## F. Shortcut / Tool Switching

- `v` で `Select`、`g` または `p` で `Pen` に切り替わる。
- `Alt + X` でアクティブパスを削除できる。
- 入力欄フォーカス中は、不要なショートカット発火が起きない。

## G. Save / Load

- 保存後、プロジェクト名表示と未保存フラグが期待通りに更新される。
- 読み込み後、パス・設定・再生設定が復元される。
- 変更なし状態で不要な未保存表示が出ない。

# 症状別パッチレシピ

## 1. 意図解釈がズレる

- 症状: 「ふわっと」で空間だけが大きく変わる
- 対応: 曖昧語の解釈表に「主編集対象」を1行追加する
- 例:
  - ふわっと: 時間優先、空間は補助
  - キレよく: 時間優先、空間は直線寄り

## 2. 過剰クリーンアップが出る

- 症状: 手描きのゆらぎが消える
- 対応: スタイル保持ルールに禁止条件を追加する
- 追加文テンプレート:
  - 「明示的な整形指示がない限り、局所的な不均一さを保持する」

## 3. JSONが不安定

- 症状: ハンドル項目が省略される、件数がずれる
- 対応: 出力制約を末尾で再掲して優先度を上げる
- 追加文テンプレート:
  - 「各 keyframe に sketchIn/sketchOut/graphIn/graphOut を必ず含め、未使用時は null を入れる」
  - 「suggestions は必ず3件で返す」

## 4. 3案の差分が小さい

- 症状: タイトルだけ違って中身がほぼ同じ
- 対応: バリエーション規約に差分軸を明記する
- 差分軸テンプレート:
  - 案A: 時間優先
  - 案B: 空間優先
  - 案C: バランス型

## 5. ルール同士が競合する

- 症状: 「滑らかに」が「味を残す」と衝突する
- 対応: 優先順位を1行で固定する
- 優先順位テンプレート:
  - 「スタイル保持を最優先とし、滑らか化はその範囲で実施する」

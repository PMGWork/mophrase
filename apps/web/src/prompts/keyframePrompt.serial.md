## 直列モードの方針

このプロンプトは、1リクエストで**3件の補正案をまとめて返す**直列生成モードで使用される。

- 補正案は必ず**3件**返す
- 3件は単なる強弱違いではなく、**編集メカニズム（アプローチ）を変える**
- 各案は共通プロンプトの制約（スタイル保持・フィールド制約）をすべて満たす

## バリエーション品質規約（最重要）

- UI 側で強度は連続調整できるため、3案の差は**強度の大小ではなく編集メカニズム**で作る。
- 3案の総変化量はおおむね同程度（中程度）に揃える。
- `suggestions[0..2]` に固定役割を割り当てない。どの案を `sketch/graph/both` にするかは、毎回ユーザー指示から動的に決める。
- ユーザー指示が片側のみ（例: 時間のみ）を示す場合、主対象は維持したまま別の編集メカニズムで3案を分ける。
- `modifierTarget` を機械的に 1:1:1 で配分しない。必要なら3案すべて同じ `modifierTarget` でよい。
- 「時間を伸ばす/ゆっくり」意図では、明示的な尺固定がない限り、**最低2案**で終点 `time` を入力終点より増やす（目安 +0.05 以上）。少なくとも1案は明確に伸長（目安 +0.08 以上）させる。
- 2キーフレーム + 時間意図のときは、3案すべてで終点 `time` を入力値から動かす（`time: 1` 固定を禁止）。各案の変化量は `|Δtime| >= 0.08`、そのうち1案は `+0.15` 以上の明確な伸長にする。
- タイトル文言と実データを一致させる。タイトルに「伸ばす/ゆっくり/ゆったり」を含む案で `time` が実質不変なのは不可。
- 選択区間が2キーフレームで時間意図が主のときは、3案とも空間は維持し、2点目 `time` を増やす。差分軸は `graph` ハンドルと伸ばし量に限定する。
- `graphCorner` でないキーフレームでは、`graphIn/graphOut` の180°対向（±5°）を維持する。時間意図で片側を変えるときは反対側も合わせて更新する。

### 3案の作り方

直列モードでは3件を同時に生成できる。この利点を活かし、**1件ずつ解釈スタンスを変える**ことで多様性を作る。

| 案    | 解釈スタンス       | 意味                                                                                                             |
| :---- | :----------------- | :--------------------------------------------------------------------------------------------------------------- |
| 1件目 | **素直な解釈**     | 指示をそのまま受け取り、最も直接的な編集を行う。迷ったら王道を選ぶ                                               |
| 2件目 | **意図の拡大解釈** | 1件目を踏まえ、指示が暗に求めている効果を汲み取り、より表現を押し出す方向に振る                                  |
| 3件目 | **別の切り口**     | 1・2件目とは異なる編集メカニズムで同じ指示を実現する。1・2件目が主に動かしたパラメータ群とは違う領域を主に動かす |

**ルール:**

- どのパラメータをどう動かすかは指示内容から都度判断する——事前に固定しない
- 2件目・3件目は前の案を参照し、**実際に異なる結果になっていることを確認してから出力する**
- 3案で差を作れないほど指示が限定的な場合は、その旨を `confidence` の低さで示す

### 禁止パターン

- タイトルだけ変えて、キーフレーム内容がほぼ同一
- 同一の編集を一様スケールしただけ（倍率違いのみ）
- 「弱・中・強」のような強度段階をそのまま3案にする
- 3案すべてを無難な中庸案に寄せる
- 時間を伸ばす意図なのに、3案とも終点 `time` が入力終点と同じ

### confidence の使い分け

- 指示が特定のモディファイアを明確に指している → `0.7〜1.0`
- 指示が曖昧で複数の解釈がある → `0.3〜0.6`
- 3案で `confidence` がすべて同じ値にならないようにする。解釈の確度が高い案ほど高く設定する

## 出力フォーマット（このモード専用）

共通スキーマの補正案を3件含む `suggestions` 配列を返す。各補正案の `keyframes` 構造は共通プロンプトの「出力スキーマ（補正案1件あたり）」に従う。

```json
{
  "suggestions": [
    { "title": "...", "modifierTarget": "...", "confidence": 0.0, "keyframes": [...] },
    { "title": "...", "modifierTarget": "...", "confidence": 0.0, "keyframes": [...] },
    { "title": "...", "modifierTarget": "...", "confidence": 0.0, "keyframes": [...] }
  ]
}
```
